# Sidebar Search Mode Design

**背景**
- 当前左栏只支持按时间分组浏览会话，不支持快速检索历史内容。
- 项目已有消息全文搜索能力，且 paste 外置存储正文已经纳入搜索，但前端仍停留在独立原型面板，没有接入左栏主流程。
- 用户希望在左栏“开始对话”按钮下加入搜索框，输入后直接进入搜索结果模式，并能打开会话后自动定位到命中消息。

**目标**
- 在左栏新增搜索框。
- 支持搜索：会话标题、Assistant 名称、消息正文、paste 外置正文。
- 搜索结果模式显示：会话标题、命中片段、时间。
- 点击结果后：打开对应会话，并自动滚动定位到命中消息。
- 第一版不引入语义匹配，仅做关键词 / 模糊匹配。

## 方案对比

### 方案 A：纯前端搜索
- 仅搜索当前已加载的会话标题与本地消息。
- 优点：实现快。
- 缺点：无法覆盖未加载消息和 paste 正文，不满足需求。

### 方案 B：后端正文搜索 + 前端标题/Assistant 模糊匹配（推荐）
- 后端负责消息正文与 paste 正文检索，并返回结果片段。
- 前端额外对会话标题和 Assistant 名称做轻量模糊匹配，然后统一排序。
- 优点：复用现有 FTS 基础，能覆盖 paste，复杂度可控，体验完整。
- 缺点：排序逻辑比单一路径稍复杂。

### 方案 C：语义检索 / Embedding
- 优点：支持按“意思”召回历史。
- 缺点：需要向量索引、生成与更新策略、资源开销和解释性设计，明显超出当前需求。

**最终方案**
- 采用方案 B。
- 不做语义匹配。
- 左栏在空查询时保持原会话列表，有查询时切换为搜索结果模式。

## 数据流
- 前端输入查询词后，300ms 防抖。
- 前端同时进行两类搜索：
  1. 后端 `search_sidebar_results(query)`：返回正文 / paste 命中的结构化结果，包含 `conversationId`、`messageId`、`snippet`、`createdAt`。
  2. 前端本地模糊匹配：基于已加载的 `conversations` 与 `assistants`，匹配标题和 Assistant 名称，生成会话级结果。
- 前端将两类结果合并、去重、排序后渲染。
- 点击结果时，页面层接收 `{ conversationId, messageId | null }`：
  - 若有 `messageId`，则打开会话并持续向上分页，直到消息出现在 `messages` 中，再滚动定位。
  - 若仅命中标题/Assistant，则只打开会话。

## 结果展示
- 搜索结果模式中不再显示“今天 / 昨天 / 最近 7 天”分组。
- 每条结果显示：
  - 第一行：会话标题
  - 第二行：命中片段
  - 右侧或次级行：相对时间
  - 可选：若会话绑定 Assistant，则显示弱化的 `#助手名`
- 当结果来自标题/Assistant 匹配时：
  - 片段显示标题或 `#助手名`
  - `messageId` 为空

## 自动定位
- 页面层新增“待定位消息 ID”状态。
- `handleConversationSelect` 扩展为支持可选 `messageId`。
- 若当前会话消息中找不到目标 `messageId`，则循环调用 `getMessages(...beforeMessageId)` 继续向上加载，直到：
  - 找到目标消息；或
  - `hasMoreMessages = false`
- `MessageList.svelte` 增加聚焦消息输入能力：
  - 接收 `focusedMessageId`
  - 计算目标消息在虚拟列表中的偏移并滚动到可见区
  - 为该消息增加短暂高亮效果

## 边界处理
- 搜索框清空后立即退出搜索结果模式，恢复普通会话列表。
- 搜索过程中显示“搜索中...”。
- 若无结果，显示“没有找到结果”。
- 已软删除消息不展示为搜索结果。
- paste 命中时，片段优先来自 paste 正文，而不是消息占位符文本。
- 第一版不做高级筛选、不做多字段高亮标注。

## 验证
- 后端：新增 `search_sidebar_results` 测试，覆盖普通正文命中与 paste 正文命中。
- 前端：新增左栏搜索模式契约测试，覆盖搜索框、结果模式、命中结果显示。
- 前端：新增消息定位契约测试，覆盖 `MessageList` 的 `focusedMessageId` 支持。
- 运行：`cargo test --manifest-path src-tauri/Cargo.toml`、定向 `node --test`、`pnpm run check`。

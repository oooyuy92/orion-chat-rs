# Conversation Sidebar Assistant Tag Design

**背景**
- 当前左侧会话列表只能看到标题、置顶状态和前缀标签。
- 会话已经支持绑定 Assistant，但该信息只在进入会话后更容易感知，列表扫描时不可见。

**目标**
- 在左栏会话列表中，为已绑定 Assistant 的会话显示一行次级信息。
- 展示形式为标题下方的 `#助手名`。
- 未绑定 Assistant 的会话保持现状，不增加占位。

**方案对比**

## 方案 A：标题下方常驻显示 `#助手名`（推荐）
- 优点：可扫描性最好；无需 hover；桌面和触控设备一致。
- 缺点：单个会话项高度略增。

## 方案 B：仅 hover 显示完整 Assistant
- 优点：列表更紧凑。
- 缺点：发现性差；触控设备体验不一致；无法快速扫视多个会话。

**最终方案**
- 采用方案 A。
- 第一行保留会话标题。
- 第二行显示小字号、弱化颜色的 `#助手名`，单行截断。
- 不增加 hover 依赖。

**架构与数据流**
- 仅做前端改动，不改后端返回结构。
- `ConversationList.svelte` 在现有 `listConversations()` 之外，额外调用 `listAssistants()`。
- 以前端内存映射 `assistant.id -> assistant.name` 为会话渲染副标题。
- 若会话的 `assistantId` 找不到对应 Assistant（例如 Assistant 已被删除），则不显示标签。

**边界与兼容**
- 重命名/加前缀输入态保持现有交互，不额外显示标签。
- 标签使用单行省略，避免挤压列表布局。
- 该改动不影响新建会话、置顶、删除、重命名逻辑。

**验证**
- 增加一个侧栏契约测试，确保组件源码包含：
  - Assistant 列表加载逻辑
  - `#助手名` 的次级标签渲染结构
- 运行定向 `node --test` 和 `pnpm run check`。
